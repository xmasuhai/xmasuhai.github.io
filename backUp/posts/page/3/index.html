体系课学习流程

为了应对学习内容复杂度曲线陡增，必须总结出流程方法，关乎效率，使在学习时间复杂度趋近平稳 α阶段：预览 预览大纲，将PPT全写进md笔记里，注意PPT里相关链接 预览测试题，预先记录到md笔记里，带着问题看视频 预览课程简介，收集到md笔记里 先快速看一遍视频x2，直播录屏，有打脸翻车部分，大致知道知识点的位置，一定要快进看 β阶段：视频学习 看视频 x 1.5 记笔记：细节 撕代码：copy & run 一定要倍速看视频； 回到知识点，暂停，回看； 记笔记，敲代码，总时间远远大于视频原始时长 笔记即思路 567 对应测试题，并分析每个选项，记入笔记 λ阶段：回顾复习 整理博客到各大平台，边查漏补缺和复习，拉起遗忘曲线 写博客 一边重放视频 x 1.8 一边整理笔记，查缺补漏 知识图谱（思维导图）结构 π阶段：默写&实操 画画原型图/效果图（Png/Gif）作成（PS/ Sketch /Figma） 演示代码(3处付代码链接): 纸上默写思路 口述测试题答案（作业即面试） 饥人谷 代码沙盒 代码钢笔 τ阶段:他山之石 搜索他人博客对应内容 链接有用内容，附上链接，声明版权 ψ阶段:总结&提炼 弄一套： 模板 模式 套路 路径 μ阶段:快速回顾 空闲时，听视频 x 2.
