git 六行配置

git config --global user.name 你的英文名
git config --global user.email 你的邮箱
git config --global push.default simple
git config --global core.quotepath false
git config --global core.editor "code --wait"
git config --global core.autocrlf input

查看配置

git config --global --list
  • 注意：上面的英文名和邮箱跟 GitHub 没有关系。 可以跟 GitHub 的用户名和邮箱保持一致， 也可以不一致。我的是一致的。
  • 其中 git config –global core.editor “code –wait”用的是VSCode 也可以用Vim
  • 注意：你需要保证 code 是可以直接在命令行执行的。 如果不能执行，你需要安装 VSCode 并配置 PATH。 我（win10）的 PATH 里添加的内容是
C:~\AppData\Local\Programs\Microsoft VS Code\bin
  • Mac 的配置方法，要在iTerm2中用命令code打开VSCode的步骤
  1. 运行VS code并打开命令面板（ ⇧⌘P ）
  2. 输入 shell command 找到: Install ‘code’ command in PATH 就行了。
  • code命令还支持其他参数：Visual Studio Code入门

常见问题

  • 问：课上说 git reset –hard XXXXXX 里面的 XXXXXX 是提交号，那为什么视频后面你又写出了 git reset –hard master 这样的命令。
  • 答：请注意，说 git reset –hard XXXXXX 的意思是我们「可以」使用 XXXXXX，并没有说我们「只能」使用 XXXXXX， 显然reset 支持的参数不止 XXXXXX 这一种。

git 就是一个命令而已

Bash命令行里有很多命令，git 就是其中一个，在git-bash里运行git命令

  • 问：手贱 已经在根目录下git init了 怎办？
  • 或者对一个已经包含了git仓库的文件夹(假设该文件夹叫做TargetFile)再进行了次git init ，没多想就直接git add -A 怎办？
  • 答：取消对文件夹 git init 初始化操作

其实想要恢复很简单，在当前目录下：

rm -rf .git/
  • 删除.git文件夹就意味着桌面不再是仓库了。
  • git只会认为有.git文件夹的路径是工作 仓库。
  • -r ：递归的删除该目录下的文件夹和文件，及子目录下的文件夹和文件
  • -f ： 忽略不存在的文件
  • 将这个文件夹删除就相当于删掉了init生成的内容，直接移除git对该文件夹造成的影响，并且不会影响到该文件夹中那个已经是git仓库的文件夹的git状态

如何删除本地仓库

开始删除所创建的仓库：

1.显示所有本地分支

git branch

2.初始化本地版本库

git init

如果显示：

Reinitialized existing Git repository in C:/Users/Administrator/test1/.git/

则:

3.找到目录下.git

ls -a

显示：

./  ../  .git/

则:

4.删除

rm -rf .git

5.可以看到master分支已经删除（test1中隐藏的.git文件夹已经删除）

6.删除git之后 再去删除本地的库文件夹即可

rm -rf XXXX

简而言之，就是删除仓库文件夹下隐藏的.git文件夹即可，直接在文件夹右键删除也可以吧
